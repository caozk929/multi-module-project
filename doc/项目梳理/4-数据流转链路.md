# 数据流转链路

## 1. 业务状态流转

### 1.1 HTTP 请求处理流转

当前项目已实现的核心数据流转链路为 **HTTP 请求的完整处理流程**，包括正常响应和异常处理两条路径。

```mermaid
stateDiagram-v2
    [*] --> 请求接收: HTTP 请求到达
    请求接收 --> 参数校验: Spring MVC 处理
    参数校验 --> 参数异常: 校验失败
    参数校验 --> 业务处理: 校验通过
    业务处理 --> 业务异常: 抛出 CustomException
    业务处理 --> 系统异常: 抛出 Exception
    业务处理 --> 正常响应: 处理成功

    参数异常 --> 异常响应: GlobalExceptionHandler
    业务异常 --> 异常响应: GlobalExceptionHandler
    系统异常 --> 企微告警: QYWXUtil.sendMsg
    系统异常 --> 异常响应: GlobalExceptionHandler
    企微告警 --> [*]

    正常响应 --> [*]: BaseResp(code=200)
    异常响应 --> [*]: BaseResp(code=500/ERROR_PARAMS)
```

## 2. 核心链路时序图

### 2.1 正常请求处理链路

```mermaid
sequenceDiagram
    participant C as 客户端
    participant F as DispatcherServlet
    participant Ctrl as TestWebController
    participant DTO as HelloDto
    participant Resp as BaseResp

    C->>F: POST /web/hello<br/>{"name": "张三"}
    F->>Ctrl: hello(HelloDto)
    Ctrl->>DTO: getName()
    DTO-->>Ctrl: "张三"
    Ctrl->>Resp: BaseResp.success("Hello from Web Controller for APP!")
    Resp-->>Ctrl: BaseResp{code="200", msg="操作成功", data="Hello..."}
    Ctrl-->>F: BaseResp<String>
    F-->>C: HTTP 200<br/>{"code":"200","msg":"操作成功","data":"Hello from Web Controller for APP!"}
```

### 2.2 业务异常处理链路

```mermaid
sequenceDiagram
    participant C as 客户端
    participant F as DispatcherServlet
    participant Ctrl as TestWebController
    participant EH as GlobalExceptionHandler
    participant Resp as BaseResp

    C->>F: POST /web/hello<br/>{"name": "1"}
    F->>Ctrl: hello(HelloDto)
    Ctrl->>Ctrl: name.equals("1") == true
    Ctrl-->>EH: throw CustomException("自定义异常")
    EH->>EH: log.error("服务异常: 自定义异常")
    EH->>Resp: BaseResp.failure()
    EH->>Resp: setCode("-1"), setMsg("自定义异常")
    Resp-->>EH: BaseResp{code="-1", msg="自定义异常"}
    EH-->>F: BaseResp<?>
    F-->>C: HTTP 200<br/>{"code":"-1","msg":"自定义异常","data":null}
```

### 2.3 系统异常处理链路（含企微告警）

```mermaid
sequenceDiagram
    participant C as 客户端
    participant F as DispatcherServlet
    participant Ctrl as Controller
    participant EH as GlobalExceptionHandler
    participant QYWX as QYWXUtil
    participant WX as 企业微信机器人
    participant Resp as BaseResp

    C->>F: HTTP 请求
    F->>Ctrl: 业务方法
    Ctrl-->>EH: throw Exception(未预期异常)
    EH->>EH: log.error("服务异常")
    EH->>QYWX: sendMsg("服务异常:" + 异常堆栈)
    QYWX->>QYWX: 截断消息至1800字符
    QYWX->>WX: HTTP POST Webhook<br/>{"msgtype":"text","text":{"content":"...","mentioned_list":["@all"]}}
    WX-->>QYWX: 响应
    EH->>Resp: BaseResp.failure()
    EH->>Resp: setCode("ERROR_PARAMS"), setMsg(异常消息)
    Resp-->>EH: BaseResp
    EH-->>F: BaseResp<?>
    F-->>C: HTTP 200<br/>{"code":"ERROR_PARAMS","msg":"...","data":null}
```

### 2.4 缓存数据流转链路

```mermaid
sequenceDiagram
    participant Caller as 业务调用方
    participant CCS as CustomCacheService
    participant Cache as Caffeine Cache
    participant EV as ExpirableValue

    Note over Caller,EV: 写入缓存（带过期时间）
    Caller->>CCS: put(key, value, TimeUnit.SECONDS, 10)
    CCS->>CCS: 计算过期毫秒数 expireMs
    CCS->>EV: new ExpirableValue(value, expireMs)
    CCS->>Cache: put(key, ExpirableValue)
    Cache-->>CCS: OK

    Note over Caller,EV: 读取缓存
    Caller->>CCS: get(key)
    CCS->>Cache: getIfPresent(key)
    Cache-->>CCS: ExpirableValue 或 原始值 或 null
    CCS->>CCS: 如果是 ExpirableValue 则解包 getValue()
    CCS-->>Caller: 原始值 或 null

    Note over Caller,EV: 缓存过期后
    Caller->>CCS: get(key)
    CCS->>Cache: getIfPresent(key)
    Cache-->>CCS: null (已被 Caffeine 自动驱逐)
    CCS-->>Caller: null
```

    O --> P
