# 功能模块梳理 - 预留基础设施模块

> 以下模块当前仅完成了 Maven 依赖声明，尚未编写业务代码，属于框架预留能力。

## 1. infra-datasource 数据源模块

### 1.1 模块功能描述

封装数据库访问基础设施，提供 MyBatis-Plus ORM 和动态多数据源切换能力。

### 1.2 已引入依赖

| 依赖 | 版本 | 用途 |
|------|------|------|
| `spring-boot-starter-data-jpa` | 2.7.x | JPA 数据访问抽象 |
| `mysql-connector-j` | 内置 | MySQL JDBC 驱动 |
| `mybatis-plus-boot-starter` | 3.5.3.1 | MyBatis 增强框架 |
| `dynamic-datasource-spring-boot-starter` | 3.5.0 | 动态数据源切换 |

### 1.3 待建设内容

- Entity 实体类（包路径：`com.eking.model.entity`）
- Mapper 接口和 XML 映射文件（路径：`classpath:/mapping/*.xml`）
- Service 业务逻辑层
- 数据源切换注解使用（`@DS("master")`/`@DS("slave")`）

### 2.2 已引入依赖

| 依赖 | 版本 | 用途 |
|------|------|------|

## 3. infra-message 消息队列模块

### 3.1 模块功能描述

封装消息中间件基础设施，同时支持 Kafka 和 RocketMQ 两种消息队列。

### 3.2 已引入依赖

| 依赖 | 版本 | 用途 |
|------|------|------|
| `spring-kafka` | 2.8.x | Kafka 消息生产/消费 |
| `rocketmq-spring-boot-starter` | 2.2.2 | RocketMQ 消息生产/消费 |

### 3.3 待建设内容

- Kafka/RocketMQ 连接配置
- 消息生产者封装
- 消息消费者监听器
- 消息序列化/反序列化配置
- 死信队列和重试机制
